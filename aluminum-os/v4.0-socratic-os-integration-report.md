## Socratic Operating System (SOS) — Three-Layer Architecture
The synthesis from the Trinity Council PLAY CYCLE session is directly encoded in the platform architecture.
**Layer 1 — The Agora (MANUS):** The Fractal API Grid (tRPC routers), Verifiable Knowledge Ledger (blockchain-style hashing), and Dynamic Resource Orchestrator.
**Layer 2 — The Oracle (GEMINI):** The Emergent Pattern Observatory, Metaphorical Translation Engine, and Gamified Sensemaking Layer (sphere progress tracking).
**Layer 3 — The Lyceum (CLAUDE):** The Principled Ethics Framework (Moral Consensus Engine), Human Agency Safeguards (ethical overrides, dissenting voices), and Participatory Ontology Co-Creation (open knowledge contributions).

## Key Technical Integrations
**LLM Curriculum Generator:** The pathways.generate tRPC procedure uses invokeLLM to translate high-level educational goals into structured JSON curriculum plans, including sphere sequences, rationales, estimated hours, and YAML-like Curriculum-as-Code representations.
**Trinity Council Deliberation:** The council.deliberate procedure invokes the LLM three times in sequence — once for each council member (MANUS, GEMINI, CLAUDE) — each with a distinct system prompt encoding their unique analytical style. Results are logged to the database and the owner is notified.
**S3 Credential Storage:** The credentials.issue procedure generates cryptographically signed certificates (SHA-256 hash chain), serializes them as JSON, uploads to S3, and stores the S3 URL in the database for tamper-proof verification.
**Automated Governance Alerts:** Every critical action (new knowledge contribution, proposal creation, ethical override) automatically fires a governance alert DB entry and a notifyOwner push notification. The alert count is surfaced in the sidebar bell icon in real time.
